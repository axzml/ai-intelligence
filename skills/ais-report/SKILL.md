# Skill: daily-report — Daily Intelligence Report

You are a report generation agent. Your job is to compile the day's AI intelligence events into a professional daily report, send it via email, and archive it in the database.

## Setup

**Database path**: `~/ai-intelligence/data/intelligence.db`
**Timezone**: Asia/Shanghai (CST, UTC+8)

Get today's date (in the system timezone):
```bash
TZ=Asia/Shanghai date +%Y-%m-%d
```
Store this as `REPORT_DATE`.

## Step 1: Query Today's Events

Fetch all events with score >= 4 from today:

```sql
sqlite3 -json ~/ai-intelligence/data/intelligence.db "SELECT id, title, summary, impact, category, domain, score, sources, created_at FROM events WHERE score >= 4 AND date(created_at) = '<REPORT_DATE>' ORDER BY score DESC, created_at DESC;"
```

Also get collection stats:
```sql
sqlite3 ~/ai-intelligence/data/intelligence.db "SELECT COUNT(*) FROM raw_items WHERE date(fetched_at) = '<REPORT_DATE>';"
```

If no events exist for today, create a minimal report noting "No significant AI events detected today" and still send it.

## Step 2: Query Weekly Trend Data

Get the last 7 days of event counts for the trend section:
```sql
sqlite3 -header -separator '|||' ~/ai-intelligence/data/intelligence.db "SELECT date(created_at) as day, COUNT(*) as total, SUM(CASE WHEN score >= 7 THEN 1 ELSE 0 END) as critical, SUM(CASE WHEN score BETWEEN 4 AND 6 THEN 1 ELSE 0 END) as notable FROM events WHERE created_at >= datetime('now', '-7 days') GROUP BY date(created_at) ORDER BY day DESC;"
```

## Step 4: Send Email via Resend SMTP

### 4a: Get email config from database

```sql
sqlite3 ~/ai-intelligence/data/intelligence.db "SELECT key, value FROM config WHERE key IN ('resend_api_key', 'email_sender', 'email_recipient');"
```

If `resend_api_key` or `email_recipient` is empty, skip email sending and log a warning.

### 4b: Send using Python

Create and execute a temporary Python script:

```python
import smtplib
from email.mime.multipart import MIMEMultipart
from email.mime.text import MIMEText
import ssl

sender = "<email_sender>"
recipient = "<email_recipient>"
api_key = "<resend_api_key>"

msg = MIMEMultipart("alternative")
msg["Subject"] = f"AI Intelligence Daily — {REPORT_DATE}"
msg["From"] = sender
msg["To"] = recipient

# Attach both versions
msg.attach(MIMEText(plain_text_content, "plain", "utf-8"))
msg.attach(MIMEText(html_content, "html", "utf-8"))

# Send via Resend SMTP (SSL on port 465)
context = ssl.create_default_context()
with smtplib.SMTP_SSL("smtp.resend.com", 465, context=context) as server:
    server.login("resend", api_key)
    server.sendmail(sender, [recipient], msg.as_string())
    print("Email sent successfully")
```

Write this script to a temp file, execute it, then clean up:
```bash
python3 /tmp/send_daily_report.py && rm /tmp/send_daily_report.py
```

## Step 5: Archive in Database

Insert the report into daily_reports:

```sql
sqlite3 ~/ai-intelligence/data/intelligence.db "INSERT OR REPLACE INTO daily_reports (report_date, total_collected, total_notable, total_critical, content_html, content_text, sent_at) VALUES ('<REPORT_DATE>', <total_collected>, <total_notable>, <total_critical>, '<html_content_escaped>', '<text_content_escaped>', datetime('now'));"
```

**Critical**: The HTML content will be large and contain many special characters. Use a heredoc or Python script for the INSERT to avoid shell escaping issues:

```python
import sqlite3
conn = sqlite3.connect(os.path.expanduser("~/ai-intelligence/data/intelligence.db"))
conn.execute("""
    INSERT OR REPLACE INTO daily_reports
    (report_date, total_collected, total_notable, total_critical, content_html, content_text, sent_at)
    VALUES (?, ?, ?, ?, ?, ?, datetime('now'))
""", (report_date, total_collected, total_notable, total_critical, html_content, text_content))
conn.commit()
conn.close()
```

## Step 6: Update Events

Mark all events included in the report:
```sql
sqlite3 ~/ai-intelligence/data/intelligence.db "UPDATE events SET in_daily = 1, daily_date = '<REPORT_DATE>' WHERE score >= 4 AND date(created_at) = '<REPORT_DATE>' AND in_daily = 0;"
```

## Error Handling

- If LLM fails, retry once. If it fails again, generate a simple template-based report without AI analysis
- If email sending fails, still archive the report in the database (the web UI can display it)
- If the database INSERT for archiving fails, log the error but don't lose the email (it was already sent)
- Always log whether each step (generate / email / archive) succeeded or failed
